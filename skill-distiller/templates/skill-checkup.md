# {{Skill 名称}} — 体检报告

## 怎么用

直接说：**"给{{skill-name}}做个体检"**

我会自动读取这个 Skill 的使用日志，分析以下内容：

---

## 体检流程

### 1. 读日志

读取 `references/usage-log.md`，统计：
- 总使用次数
- 使用频率（每天/每周/每月多少次）
- 最近一次使用时间

### 2. 找问题

从日志的"问题"字段中提取：
- 出现过的问题列表（去重）
- 哪些问题反复出现
- 哪些问题已经解决

### 3. 算价值

根据使用频率和问题情况，判断这个 Skill：
- **高频且稳定** → 保持现状，定期维护
- **高频但有问题** → 优先修复，减少每次使用的问题
- **低频且稳定** → 考虑合并到其他 Skill 或归档
- **低频且有题** → 评估是否值得修，还是重新做一个

### 4. 出报告

```
╔══════════════════════════════════════╗
║     {{Skill 名称}} 体检报告          ║
╠══════════════════════════════════════╣
║ 使用次数：XX 次                      ║
║ 使用频率：约 X 次/周                 ║
║ 最近使用：YYYY-MM-DD                 ║
╠══════════════════════════════════════╣
║ 状态：🟢 健康 / 🟡 需关注 / 🔴 需修复  ║
╠══════════════════════════════════════╣
║                                      ║
║ ✅ 做得好的：                        ║
║ - ...                                ║
║                                      ║
║ ⚠️ 需要改进的：                      ║
║ - ...                                ║
║                                      ║
║ 💡 建议操作：                        ║
║ 1. ...                               ║
║ 2. ...                               ║
║                                      ║
╚══════════════════════════════════════╝
```

### 5. 问用户要不要改

> 体检完了。最需要改的是 {{最重要的建议}}。
> 你要现在就改，还是先记下来以后再说？

如果用户选择改，执行改进并更新版本号。

## 用完记一笔

每次用完这个 Skill，在 `references/usage-log.md` 末尾追加一条记录：

```markdown
## YYYY-MM-DD HH:MM | 第N次使用
- **任务**：一句话描述做了什么
- **输入**：用户给了什么
- **结果**：输出是什么
- **问题**：有没有哪里不好用/出错/需要改（没有就写"无"）
```
