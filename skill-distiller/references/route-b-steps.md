# 路线B：把步骤串起来

**适用场景**：用户每次做这件事都按差不多的步骤来，步骤固定、重复性高

**跟用户怎么说**：
> 你这个活儿每次步骤都差不多，最好的办法是把步骤列出来让 AI 按顺序跑。你现在在心里过一遍，每次做这个活儿，你是按什么顺序来的？

---

## 引导步骤

### B1. 列出步骤

> 你每次做这个活儿，第一步干嘛、第二步干嘛...能不能列一下？

**用户可能会**：
- 顺利列出来 → 直接记下来
- 列得太粗（比如"处理一下"）→ 追问："处理一下具体是做什么？"
- 列得太细（比如"打开浏览器→输入网址→..."）→ 帮他合并："这几步可以合为一步：从网站抓取内容。"

**如果用户列不清楚**：
> 没关系，你随便想一个最近的例子，从头到尾跟我讲一遍你当时是怎么做的，我帮你提炼步骤。

### B2. 搞清每一步的输入输出

对每一个步骤，问清楚：

> 第X步"【步骤名】"：
> - 这一步开始的时候，你手上有什么？（输入）
> - 这一步做完之后，你得到了什么？（输出）
> - 你是手动做的还是用了什么工具？

把每个步骤的输入输出画成一条链：

```
输入 → [步骤1] → 中间结果1 → [步骤2] → 中间结果2 → ... → 最终输出
```

给用户看一遍："是这样的流程吗？"

### B3. 找出可以自动化的环节

分析每个步骤：
- **纯体力活**（复制粘贴、格式转换、整理排版）→ 100% 可以自动化
- **需要判断但标准明确**（分类、筛选、按规则填写）→ 可以自动化，AI 按规则来
- **需要创意或复杂决策**（写文案、设计方案）→ 半自动化，AI 出初稿人来改

告诉用户：
> 我看了你的步骤，【这几步】可以完全交给 AI，【这步】AI 出个初稿你来定，【这步】暂时还得到你手动来。

### B4. 先做单步，再串联

> 建议：我们先做一个最简单的版本——只自动化其中一步。跑通了再串下一步。一步步来不容易翻车。

让用户选最容易、最常用的一步先做。

做完第一步的 Skill → 试跑 → 用户满意 → 再做下一步 → 串联。

### B5. 串联成完整流程

当多个单步 Skill 都做好了之后，生成一个"总管" Skill：

```
## 完整流程
按顺序执行以下 Skill：
1. 调用 [步骤1-Skill名]，输入：xxx
2. 把步骤1的输出作为输入，调用 [步骤2-Skill名]
3. 把步骤2的输出作为输入，调用 [步骤3-Skill名]
...
```

告诉用户：
> 以后你只需要说"处理一下【任务名】"，AI 就会从头跑到尾。

---

## 常见问题

**Q：有些步骤需要等外部工具的结果**
> 没问题，可以在 Skill 里写"先调用xxx API，等结果返回后再继续"。你需要提供 API 的调用方式。

**Q：有些步骤不一定每次都要**
> 可以在 Skill 里加判断："如果用户说不需要xxx，就跳过第X步。" 让 AI 灵活处理。

**Q：步骤之间的数据怎么传递？**
> 最简单的方式是让 AI 把每一步的结果放在一个临时文件里，下一步从文件里读。我在 Skill 模板里会帮你处理这个。

---

## 路线B核心原则

- **先跑通一步，再串下一步**。不要一上来就做完整流程，容易出错不好排查
- **每一步的输入输出必须明确**。模糊的步骤 = 出不稳定的 Skill
- **串联后整体试跑 3 次**。连续 3 次不用手动改，才算稳定
